# コーヒー焙煎待ち時間表示システム

小規模コーヒー焙煎所向けの、公開画面・管理画面・履歴画面を備えた初期版です。

## ファイル構成

- `index.html` 公開画面
- `admin.html` 管理画面
- `history.html` 履歴画面
- `style.css` 共通スタイル
- `app.js` 共通ロジック
- `public.js` 公開画面処理
- `admin.js` 管理画面処理
- `history.js` 履歴画面処理
- `firebase-config.js` Firebase設定
- `firebase.rules.json` Realtime Database ルール例
- `sample-data.json` 初期データ例

## 事前準備

1. Firebaseで新しいプロジェクトを作成する
2. Realtime Database を作成する
3. 開発時はテストモードで開始する
4. `firebase-config.js` のプレースホルダを実際の値に置き換える
5. 必要に応じて `sample-data.json` を Realtime Database に投入する

## 初期設定値

- 1件あたり加算時間: 15分
- 減少開始ラグ: 5分
- 減少刻み時間: 5分
- 4桁PIN: 1234

## 待ち時間ロジック

1件ごとの待ち時間推移は以下です。

- 注文から0分以上10分未満: 15分
- 注文から10分以上15分未満: 10分
- 注文から15分以上20分未満: 5分
- 注文から20分以上: 0分

現在の待ち時間は、`active` な注文の現在寄与分を合計して算出します。

## 管理画面の操作

- 注文追加: 新規注文を1件追加
- 取消: 直前の有効注文のうち、まだタイマー残存中のものだけ取消
- 1件戻す: 最も古い有効注文1件を完了扱い
- 受付開始 / 受付終了: 公開画面の状態を切替
- 設定保存: 待ち時間パラメータを更新

## 公開前の注意

`firebase.rules.json` は動作確認用に読み書き開放しています。
本番公開前には、少なくとも書き込み制限を見直してください。

また、4桁PINは軽い入室制御であり、厳密な認証ではありません。
必要に応じて後段で認証方式を強化してください。

## GitHub Pages への配置

1. このファイル一式をリポジトリに配置する
2. GitHub Pages を有効化する
3. `firebase-config.js` を設定済みの状態で push する
4. `index.html` を公開画面として利用する
5. `admin.html` を管理画面、`history.html` を履歴画面として利用する
